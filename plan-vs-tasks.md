# Claude Code: Plan 模式 vs Tasks/Todos 模式

## Plan 模式

### 特点
- **目的**：在开始实现复杂任务前，先**探索代码库**并**制定实施计划**
- **流程**：
  1. 探索代码库结构
  2. 理解现有模式和架构
  3. 设计实施方法
  4. 展示计划给用户审批
  5. 获得批准后执行
- **适用场景**：
  - 新功能实现（需要架构决策）
  - 代码重构（影响范围大）
  - 复杂的多步骤任务
  - 需要用户选择技术方案的情况

### 工具权限
- 可以使用 **Explore**、**Glob**、**Grep**、**Read** 等只读工具
- **不能**使用 **Edit**、**Write**、**Bash**（除 git）等修改工具

### 示例
```
用户：添加用户认证功能
Claude：进入 Plan 模式 → 探索代码 → 设计认证方案 → 用户确认 → 执行
```

---

## Tasks/Todos 模式

### 特点
- **目的**：**跟踪任务进度**和**管理工作流程**
- **流程**：
  1. 创建任务列表
  2. 分配任务给团队成员
  3. 标记任务状态
  4. 跟踪依赖关系
- **适用场景**：
  - 团队协作
  - 复杂项目的任务分解
  - 需要并行处理的任务
  - 需要跟踪进度的项目

### 工具权限
- 拥有所有工具的完整权限
- 可以直接修改代码、执行命令等

### 示例
```
用户：实现一个新功能
Claude：创建任务列表 → TaskCreate → 分配给不同 agent → 跟踪进度
```

---

## 对比总结

| 特性 | Plan 模式 | Tasks/Todos 模式 |
|------|-----------|-----------------|
| **主要目的** | 设计和规划 | 任务管理和跟踪 |
| **工具权限** | 只读（探索） | 完整（可修改） |
| **交互方式** | 用户审批计划 | 用户/agent 协作 |
| **适用时机** | 开始实现前 | 项目进行中 |
| **团队协作** | 单人规划 | 多人协作 |
| **EnterPlanMode** | 需要调用 | 不需要 |

## 典型使用场景

### 使用 Plan 模式的情况
- "我需要添加一个新功能，应该怎么设计？"
- "这个代码需要重构，应该从哪里开始？"
- "有几种实现方案，哪个更好？"

### 使用 Tasks/Todos 模式的情况
- "这个项目有 5 个功能要实现"
- "需要团队协作开发"
- "需要跟踪哪个功能完成了"
